# Tech Reference: Monitoring Observability

| Name | Description | URL |
|------|-------------|-----|
| **Sentry** | The standard for application error monitoring. Real-time crash reports | sentry.io |
| **Grafana** | The open-source observability platform. Connect to Prometheus, Loki, T | grafana.com |
| **Prometheus** | Open-source systems monitoring and alerting toolkit. Pull-based metric | prometheus.io |
| **OpenTelemetry** | Vendor-neutral observability framework for traces, metrics, and logs.  | opentelemetry.io |
| **PostHog** | Open-source product analytics you can self-host. Feature flags, sessio | posthog.com |
| **Loki** | Grafana's Prometheus-inspired log aggregation system - index labels, n | grafana.com/oss/loki |
| **Tempo** | Grafana Tempo - high-scale distributed tracing backend. | grafana.com/oss/tempo |
| **Jaeger** | CNCF distributed tracing - UI for service maps and root-cause analysis | www.jaegertracing.io |
| **Zipkin** | Twitter's original distributed tracing system - predates OpenTracing. | zipkin.io |
| **Datadog** | Comprehensive SaaS observability - APM, logs, infra, security. | www.datadoghq.com |
| **New Relic** | Veteran APM SaaS - full-stack observability with One platform. | newrelic.com |
| **Dynatrace** | Enterprise APM/observability - AI-driven root cause (Davis). | www.dynatrace.com |
| **Splunk** | Big-data log analytics - the on-prem incumbent acquired by Cisco. | www.splunk.com |
| **Honeycomb** | Charity Majors' observability platform - high-cardinality event analys | www.honeycomb.io |
| **Signoz** | Open-source full-stack observability on ClickHouse - APM + logs + trac | signoz.io |
| **Uptime Kuma** | Self-hosted uptime monitoring - beautiful UI, status pages. | github.com/louislam/uptime-kuma |
