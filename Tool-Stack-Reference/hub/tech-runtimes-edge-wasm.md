# Tech Reference: Runtimes Edge Wasm

| Name | Description | URL |
|------|-------------|-----|
| **Deno** | Secure JS/TS runtime by Node's creator - built-in TS, ESM, permissions | deno.com |
| **Node.js** | The JavaScript runtime that started server-side JS - V8 + libuv. | nodejs.org |
| **Cloudflare Workers** | Edge compute on Cloudflare's network - V8 isolates, sub-ms cold start. | workers.cloudflare.com |
| **Deno Deploy** | Deno's global edge platform - deploy TS to 35+ locations. | deno.com/deploy |
| **WinterJS** | Wasmer-built Service Worker-compatible JS runtime at the edge. | wasmer.io/products/winterjs |
| **Wasmtime** | Bytecode Alliance Wasm runtime - fast, standards-compliant. | wasmtime.dev |
| **Wasmer** | Universal Wasm runtime - run Wasm anywhere with WASIX extensions. | wasmer.io |
| **WASI** | WebAssembly System Interface - sandboxed system calls for Wasm. | wasi.dev |
| **WebAssembly** | Portable binary instruction format - runs at near-native speed in brow | webassembly.org |
| **Spin** | Fermyon's serverless framework for Wasm components. | www.fermyon.com/spin |
| **Fastly Compute** | Fastly's Wasm-based edge compute platform. | www.fastly.com/products/edge-compute |
| **Vercel Edge Runtime** | Vercel's Web-standard edge runtime - Next.js middleware and edge funct | edge-runtime.vercel.app |
| **Netlify Edge Functions** | Netlify's edge runtime - based on Deno. | docs.netlify.com/edge-functions/overview |
| **Fresh** | Deno-native web framework - islands architecture, edge by default. | fresh.deno.dev |
